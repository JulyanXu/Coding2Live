# Writing and Publishing

## Audience

People writing blogs, reports, ebooks, course notes, or long-running publishing projects.

## Problems Solved

- Select topics and reader promises from a messy idea pool.
- Organize materials into outlines, chapters, and editing checklists.
- Convert Markdown into web pages, PDFs, or ebook formats.

## Vibe Coding Workflow

1. Define reader, goal, tone, and publishing channel.
2. Ask AI to draft an outline and objections list from source material.
3. Write the key arguments and cases yourself, then use AI for structural editing.
4. Use Markdown and Pandoc for repeatable publishing.
5. Record feedback after release and update the next version.

## Recommended Tools

- Markdown and GitHub for versioned writing.
- Pandoc for format conversion.
- A style guide for citations, headings, and punctuation.

## Example Prompt

```text
Act as a structural editor. Based on these notes, plan a 2,500-word article:
1. Reader: non-technical team leads.
2. Topic: improving weekly reports with AI.
3. Output title options, core argument, section outline, and examples.
4. Mark paragraphs that require sources.
5. Do not draft the full article yet; improve the structure first.
```

## Reference Project

Manage Markdown drafts in GitHub, export formats with Pandoc, and maintain a source list per chapter.

## Limitations and Risks

AI may produce generic prose or fabricated sources. Facts, citations, cases, and opinions must be verified by the author.
