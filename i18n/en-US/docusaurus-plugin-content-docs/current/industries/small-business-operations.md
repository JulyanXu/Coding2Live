# Small Business Operations

## Audience

Small businesses without a dedicated technical team that still need better order handling, customer service, inventory, scheduling, and reporting.

## Problems Solved

- Automate repetitive replies, inventory checks, order summaries, and daily reports.
- Turn oral processes into SOPs.
- Test whether an internal tool is worth building.

## Vibe Coding Workflow

1. List operational tasks repeated more than three times per week.
2. Break each task into trigger, input, process, output, and owner.
3. Ask AI to draft SOPs, spreadsheet fields, and automation flows.
4. Validate first with spreadsheets, Notion, Zapier, or similar tools.
5. Build scripts or internal tools only after the process stabilizes.

## Recommended Tools

- Google Workspace for forms, sheets, and shared documents.
- Notion for SOPs, customer notes, and project boards.
- Zapier for connecting forms, email, spreadsheets, and notifications.

## Example Prompt

```text
Act as a small-business operations consultant. Turn this manual process into a lightweight automation plan:
1. Business: local bakery receiving orders through chat and forms.
2. Pain points: slow inventory checks, repeated replies, manual daily sales summaries.
3. Output spreadsheet fields, SOP, automation triggers, and daily-report fields.
4. Provide a no-code validation version and a future custom-app version.
```

## Reference Project

Use forms to collect orders, spreadsheets to summarize them, and Zapier to send reminders and daily digests.

## Limitations and Risks

Do not overbuild early. Small-business processes change quickly; stable fields and ownership matter more than code at first.
