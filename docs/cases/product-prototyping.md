# 案例：产品原型

## 适用人群

需要快速验证产品想法和关键交互的人。

## 可解决的问题

需求模糊、页面状态遗漏、原型无法验收。

## Vibe Coding 工作流

用 AI 输出用户故事、页面清单和关键状态，再生成原型并用 Playwright 验证路径。

## 推荐工具

Figma、GitHub Copilot/Codex、Playwright。

## 示例提示词

```text
请把这个产品想法转成 MVP 原型规格：用户故事、页面、状态、验收路径和不做事项。
```

## 真实/参考项目

先做 3 天内能完成的静态交互原型，再决定是否接入后端。

## 限制与风险

不要把原型阶段扩张成完整产品开发。
