# 写作与出版

## 适用人群

适合需要写博客、报告、电子书、课程讲义或长期出版项目的人。

## 可解决的问题

- 从主题池筛选选题和读者承诺。
- 把素材整理成大纲、章节和编辑清单。
- 将 Markdown 转换成网页、PDF 或电子书格式。

## Vibe Coding 工作流

1. 明确读者、写作目标、语气和发布渠道。
2. 让 AI 根据素材生成大纲和反对意见清单。
3. 人工完成关键观点和案例，再让 AI 做结构编辑。
4. 使用 Markdown 和 Pandoc 维护可重复出版流程。
5. 发布后记录反馈，更新下一版。

## 推荐工具

- Markdown 和 GitHub，用于版本管理和协作。
- Pandoc，用于格式转换。
- 风格手册，用于统一引用、标题和标点规则。

## 示例提示词

```text
你是结构编辑。请基于下面素材帮助我规划一篇 2500 字文章：
1. 读者：非技术背景的团队负责人。
2. 主题：如何用 AI 改善周报流程。
3. 输出：标题备选、核心论点、章节大纲、每节关键例子。
4. 标出哪些段落需要事实来源。
5. 不要代写全文，先帮我优化结构。
```

## 真实/参考项目

可以用 GitHub 管理 Markdown 草稿，用 Pandoc 导出不同格式，并为每个章节维护来源清单。

## 限制与风险

AI 可能产生陈词滥调或虚构来源。引用、事实、案例和个人观点必须由作者本人确认。
